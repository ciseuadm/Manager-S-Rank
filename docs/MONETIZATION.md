# 💰 MONETIZATION — реклама, Мана-руда, платежи

> Главная идея: бот бесплатен для чатов, но **окупает сервер и приносит доход**
> через ненавязчивую рекламу и внутреннюю экономику.

---

## 1. Источники дохода (обзор)

| Канал дохода | Как работает | Кто платит |
|---|---|---|
| **Реклама в чатах** | 1 пост/день во все чаты, где бот админ | рекламодатели / ты |
| **Реклама в канале бота** | посты в твоём канале-витрине | рекламодатели |
| **Продажа Мана-руды** | пакеты руды за Telegram Stars | пользователи |
| **Продажа рекламных мест** | рекламодатель покупает кампанию за Stars | рекламодатели |
| **Premium-фичи чатам** | расширенная статистика/настройки за Stars | админы чатов |
| **Комиссия с обмена** | при обмене руды на подарки берём спред | пользователи |

Telegram **Stars (XTR)** — нативная валюта Telegram для цифровых товаров.
Боты принимают Stars без внешнего эквайринга. Это основной денежный поток.

## 2. Рекламная подсистема

### 2.1 Поток (как ты отправляешь рекламу)

```
Ты в личке боту → /newad → бот спрашивает:
  1) контент (текст/фото/видео или forward готового поста)
  2) кнопку (текст + ссылка) — опционально
  3) таргет: все чаты / только канал / выбор
  4) сколько дней крутить
→ создаётся ad_campaign (status=active)
→ APScheduler каждый день в ADS_SEND_HOUR рассылает активные кампании
→ в каждый чат не больше ADS_DAILY_LIMIT_PER_CHAT (=1) поста в день
→ пишется ad_impressions, days_done++; когда days_done==days_total → status=done
```

### 2.2 Анти-спам гарантии

- Жёсткий лимит: **1 рекламный пост в день на чат** (проверка по `ad_impressions`).
- Рассылка через `services/broadcaster.py`: задержка ~0.05с между чатами,
  обработка `RetryAfter`, пропуск чатов, где бот забанен/удалён (авто-чистка).
- Админ чата может отключить рекламу (`/ads off`) — но тогда теряет часть
  бесплатных премиум-функций (мягкое стимулирование оставлять рекламу включённой).
- Реклама визуально помечается («📢 Реклама») — честно и по правилам Telegram.

### 2.3 Статистика

- На кампанию: охват (сколько чатов), доставлено/ошибок, по дням.
- Глобально в панели владельца: суммарные показы, активные кампании, доход.

### 2.4 Рекламный кабинет (на будущее)

Рекламодатель сам создаёт кампанию в личке бота, платит Stars, кампания уходит
на модерацию владельцу (`status=pending` → ты подтверждаешь → `active`).

## 3. Экономика Мана-руды

Мана-руда (Mana / руда) — игровая валюта в духе Solo Leveling: охотники
«добывают» её активностью, как из подземелий.

### 3.1 Как зарабатывать (источники)

| Действие | Награда (дефолт, настраивается) |
|---|---|
| Сообщение в чате | +1 руда (с антифлуд-кулдауном) |
| Ежедневный бонус `/daily` | +50 руды |
| Приглашённый друг (реферал) | +200 руды |
| Достижение нового ранга | разовый бонус (E→D +100 … S +5000) |
| Топ недели в чате | бонус 1/2/3 место |
| Выполнение реф-цели | бонус по `referral_goals` |

Антифрод: кулдаун на руду за сообщения, защита от самоприглашений
(нельзя пригласить себя/уже состоящего), журнал `mana_tx` для аудита.

### 3.2 Куда тратить (магазин `/shop`)

| Товар | Цена (пример) | Что даёт |
|---|---|---|
| Снятие своего мута | N руды | разблокировка раньше |
| Буст ранга (x2 руда на час) | N руды | ускорение прокачки |
| Кастомное звание/титул | N руды | косметика в карточке |
| Реклама своего поста в чате | N руды | 1 показ в своём чате |
| **Подарок Telegram** | много руды | бот дарит gift через API |
| **Telegram Premium** | очень много руды | оплата Premium получателю |

### 3.3 Обмен руды на реальные блага

- **Telegram-подарки**: бот покупает gift за **свои Stars** и дарит пользователю,
  списывая эквивалент в руде. Курс задаёт владелец (спред = доход).
- **Telegram Premium**: бот оплачивает Premium через `giftPremiumSubscription`.
- Курс руда↔Stars устанавливается так, чтобы покрывать расходы + маржа.
- Лимиты и анти-абуз: дневные лимиты вывода, ручное подтверждение крупных обменов.

### 3.4 Перевод между игроками `/transfer`

Игроки могут дарить руду друг другу (с маленькой комиссией в «казну», что создаёт
дефляцию и доп. ценность). Через `mana_tx` обе стороны.

## 4. Платежи через Telegram Stars

### 4.1 Что продаём за Stars

- **Пакеты Мана-руды**: 100⭐ = N руды, 500⭐ = N+бонус и т.д.
- **Рекламные кампании**: цена = дни × тариф.
- **Premium-фичи чату**: расширенная аналитика, больше реф-целей, кастом-брендинг.

### 4.2 Реализация (aiogram)

- `bot.send_invoice(..., currency="XTR", prices=[LabeledPrice(...)])`
- Обработать `pre_checkout_query` → `answer_pre_checkout_query(ok=True)`.
- На `successful_payment` → начислить товар, записать в `payments`.
- Рефанды: `refundStarPayment` по `telegram_charge_id`.

### 4.3 Юридич./правила

- Stars выводятся в реальные деньги через Telegram (Fragment) — это легальный путь.
- Не обещать пользователям вывод фиата за руду — только цифровые блага.
- Соблюдать [Bot Payments](https://core.telegram.org/bots/payments-stars) и правила
  рекламы Telegram.

---

## 5. Экономический баланс (чтобы не разориться)

Золотое правило: **стоимость выдаваемого блага (в Stars) ≤ доход от пользователя**.

- Курс руды подбирается так, что «бесплатно» намайнить дорогой подарок очень долго,
  а купить за Stars — быстро. Майнинг = удержание, Stars = доход.
- Все расходные операции (подарки/Premium) — только при достаточной казне Stars.
- Мониторинг: панель владельца показывает эмиссию руды, расход Stars, P&L.

---

## 6. Метрики для отслеживания

- DAU/MAU, число чатов, удержание (retention по `/daily`).
- Виральность: коэффициент приглашений (k-factor).
- Доход: Stars/день, ARPU, конверсия в покупки.
- Реклама: показы/день, доля чатов с включённой рекламой.
