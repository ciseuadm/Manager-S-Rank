# 🔌 Как подключить авто-проверку заданий (для партнёров)

> Эту страницу можно целиком отправлять рекламодателю/партнёру. Здесь объяснено,
> что и зачем делать, с готовым кодом. После настройки выполнение заданий
> засчитывается **автоматически**, без скриншотов и ручных подтверждений.

---

## 1. О чём речь простыми словами

Наша система S-Rank приводит вам живых пользователей за задания (подписка на
канал, запуск бота, действие в боте/игре). Чтобы начислить пользователю награду,
системе нужно **достоверно узнать, что он выполнил условие**.

Есть два случая:

- **Подписка на канал** — ничего настраивать не нужно: просто добавьте нашего
  бота администратором канала (право «читать участников»). Подписку система
  проверяет сама.
- **Действие в боте/игре** (запустил бота, набрал N очков, прошёл шаг и т.п.) —
  Telegram **не позволяет** одному боту «заглянуть» в другой. Поэтому используется
  стандартная схема **постбэк** (server-to-server callback, как в CPA-сетях):
  ваш бот сам сообщает нам «пользователь выполнил» одним HTTP-запросом.

Вы **не передаёте нам код и доступы**. Только добавляете один вызов в свой бот.

---

## 2. Как это работает (3 шага)

```
1) Пользователь в S-Rank жмёт задание и переходит в ваш бот по ссылке:
        https://t.me/ВАШ_БОТ?start=<TOKEN>
   TOKEN — это подписанный нами разовый код пользователя (мы выдаём его сами).

2) Ваш бот в обработчике /start читает этот TOKEN (это «start payload»)
   и запоминает его за пользователем.

3) Когда пользователь выполнил условие задания, ваш бот делает один запрос:
        GET https://НАШ_АДРЕС/cb/task?token=<TOKEN>

   Мы проверяем подпись TOKEN, понимаем какой это пользователь и какое задание,
   и сами начисляем награду. Готово.
```

Точную ссылку `https://НАШ_АДРЕС/cb/task...` и пример вы получаете от нас (команда
`/taskcb <id задания>` у владельца системы).

---

## 3. Что нужно сделать вам (один раз)

1. Принять «start payload» в обработчике `/start` и сохранить TOKEN за пользователем.
2. В момент выполнения условия — вызвать наш URL с этим TOKEN.

Это всё. Дальше система начисляет награду сама.

---

## 4. Готовый код

### Python (aiogram 3.x)

```python
import aiohttp
from aiogram.filters import CommandStart, CommandObject

CB_URL = "https://НАШ_АДРЕС/cb/task"   # пришлём вам
_tokens: dict[int, str] = {}            # user_id -> token (лучше хранить в БД)

@dp.message(CommandStart(deep_link=True))
async def on_start(message, command: CommandObject):
    token = command.args                # это наш TOKEN из ссылки ?start=...
    if token:
        _tokens[message.from_user.id] = token
    # Если задание = «просто запустить бота» — условие уже выполнено, рапортуем:
    await report_done(message.from_user.id)
    await message.answer("Привет! 👋")

async def report_done(user_id: int):
    token = _tokens.get(user_id)
    if not token:
        return
    async with aiohttp.ClientSession() as s:
        await s.get(CB_URL, params={"token": token})
```

Для задания «набрать 20 очков» вызовите `await report_done(user_id)` в тот момент,
когда счёт игрока достиг 20.

### Любой язык / curl

```bash
curl "https://НАШ_АДРЕС/cb/task?token=ТОКЕН_ПОЛЬЗОВАТЕЛЯ"
```

Метод — `GET` или `POST` (тело JSON `{"token": "..."}` тоже принимается).

## 5. Частые вопросы

**Это безопасно? Не начислят ли награду «накрутчики»?**
Нет. TOKEN подписан нашим секретным ключом и привязан к конкретному пользователю
и заданию. Подделать токен для чужого аккаунта невозможно — ваш бот лишь
возвращает тот токен, который мы выдали реальному перешедшему пользователю.

**Что если запрос отправится дважды?**
Ничего страшного — начисление идемпотентно: награда выдаётся один раз, повторные
вызовы просто игнорируются (ответ `already`).

**Что возвращает наш сервер?**
JSON, например: `{"ok": true, "code": "credited", "reward": 100}`.
Коды: `credited` (начислили), `already` (уже было), `bad_token` (неверная подпись),
`inactive` (задание выключено).

**Нужно ли отдавать вам код/токен бота/доступы?**
Нет. Только добавить один вызов URL. Канал — только добавить бота администратором.

**А если пользователь пришёл не по нашей ссылке?**
Тогда у него нет TOKEN — награды не будет (это и есть защита). Поэтому важно
принимать `start payload` и сохранять его.

## 6. Что нельзя автоматизировать

Если источник (бот/игра) **не сотрудничает** (не присылает постбэк) — Telegram не
даёт проверить выполнение «снаружи». Тогда остаётся только ручная проверка по
скриншоту. Поэтому для авто-зачёта нужен этот простой постбэк — других «волшебных»
способов читать чужие боты в Telegram нет.
