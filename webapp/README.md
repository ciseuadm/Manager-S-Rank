# Mini App (Telegram WebApp)

Витрина-«платформа» бота: кошелёк, задания с таймерами, магазин наград, топ.
Бэкенд (`services/webapp.py`) поднимается **в том же процессе**, что и бот
(aiohttp), и переиспользует `services/*`. Фронт — этот `index.html` (один файл).

## Как включить (остаётся только вписать значения)

1. Захостить бота на платформе с публичным HTTPS-доменом (например, Railway).
   На Railway порт берётся из `PORT` автоматически — менять ничего не нужно.
2. Прописать переменные окружения (`.env` или панель хостинга):

   ```env
   WEBAPP_ENABLED=1
   WEBAPP_URL=https://<твой-домен>.up.railway.app   # СЮДА ВСТАВИТЬ публичный https-адрес
   ```

3. Перезапустить бота. На старте он:
   - поднимет HTTP-сервер Mini App (отдаёт этот `index.html` на `/`);
   - поставит кнопку-меню «🎮 Платформа» рядом с полем ввода в ЛС;
   - добавит кнопку «🎮 Открыть платформу» в `/menu`.

## Проверка домена для TWA (по желанию)

Чтобы Mini App открывался по короткой ссылке `t.me/<bot>/<app>`, в @BotFather:
`/newapp` → выбрать бота → указать тот же `WEBAPP_URL`. Кнопки выше работают и
без этого шага (open via direct URL).

## Безопасность

Каждый запрос несёт Telegram `initData`, который бэкенд проверяет по HMAC с
токеном бота (`validate_init_data`) и отвергает просроченный. Это надёжно
подтверждает `user_id` — отдельные пароли/сессии не нужны.
